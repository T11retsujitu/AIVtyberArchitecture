# 03 — Game Design Playbook（起案の指針：期待ギャップ・情報非対称・判定パネル）

このドキュメントは**新しいゲームを起案するとき**に読む発想・選定の指針。
`docs/00`（キャラ）と `docs/01`（1本の夢の型）が「どう在るか・どう閉じるか」を定めるのに対し、ここは**「何を作ると面白いか」**——着眼点・起案テンプレ・判定パネル——を定める。
perceive 実装の規範（夢がどこまで嘘をつけるか）は `docs/04` が正。契約（`docs/02`/`07`/`09`）はこの doc の適用で一切変わらない。

---

## なぜこの doc があるのか（実測の教訓）

Mode B+ 実測（2026-07-02・NIM・10本）は**直接投稿可 0/10** だった。セリフはキャラらしいのに、10本全部が「見守る→震えが鎮まる→満ち」に収束し、核の緊張が一度も立たなかった（`review/synthesis/15-game-recommendation.md`・`docs/14`）。

根本原因は個別ゲームの調整不足ではなく、**視聴者が2秒で分かる「正解」がゲームに存在しない**こと。視聴者の期待が立たなければ、AIちゃんの行動との**ギャップ**も立たず、コメントの突っ込み余地（「そこはこうだろ！」）がゼロになる。既存2ゲームの action（wait / hover / lookAway / withdraw …）はすべて対象への**態度**であって、解くべき**課題**がない——これが単調さの正体。

この doc は、その反省を「起案時に必ず問う型」として固定する。

---

## 1. 中核原理 — 期待ギャップエンジン

> **人間（視聴者）ならこうする。けどAIちゃんは間違っちゃう。**

これがエンタメの中核。成立条件は3つ。起案時に**全部**を満たすこと：

| # | 条件 | 満たさないとどうなるか |
|---|---|---|
| ① | **視聴者が2秒で「自分ならこうする」を持てる**（視聴者正解の存在） | 期待が立たない＝ギャップも立たない。満ち一択・0/10 の再来 |
| ② | **AIちゃんが外す理由がキャラとして納得できる**（→ §3） | ランダムな間違いは「アホ」。筋の通った間違いは「愛おしい」 |
| ③ | **間違いの結果が画面で見える**（Tier B・不可逆な変化） | オチが伝わらない。25秒の最後が「何も起きてない」に見える |

「視聴者正解」は攻略上の勝ち筋のことではない。**中学生以上なら誰でも即答できる常識的な一手**（危ないものに近づかない・嘘は信じない・謝られたら許す…）のこと。夢のルール側でそれを裏切ってもよい（→常識反転・§4）。裏切るにしても、**まず期待を立てるのが先**。

なお「正しさより、らしさ」（`docs/00 §3`）とは矛盾しない。**AIちゃんは今後も正解を狙わない**。変わるのは設計側：正解が存在しない世界では「らしさ」はギャップとして観測できない。**正解を置くから、外したときの「らしさ」が立つ**。

25秒との関係：この形式なら25秒は短くない。「常識で分かる状況＋一つの判断＋結果＋受け止め」にちょうどいい尺。25秒が短く感じるのは、雰囲気の立ち上げだけで尺を使い切る設計のとき（それは別ジャンル＝長尺のムードピース）。

## 2. 情報非対称 — 視聴者は知ってる、AIちゃんは知らない

不変条件 #1「Perception に座標・数値を露出させない」は **AIちゃんに対して**隠すルールであって、**視聴者に対してではない**。

- **視聴者**には `RawState` の真実を見せられる（Tier B レンダー・`docs/11 §2`。リプレイで再生した生状態を描く）
- **AIちゃん**には `perceive()` の描写しか渡らない（`docs/07`）——そして描写は**夢として歪んでよい**（`docs/04`）

この非対称は**劇的アイロニー（視聴者は知ってる／本人は知らない）の生成装置**。原型は `review/Antigravity/game_ideas.md` の「背後の影」：

> 視聴者には恐ろしいバケモノが近づいている映像が見えているのに、AIちゃんは「……なんだか、後ろが重たい気がする」と呑気。

定番形として使い分ける：

- **視聴者だけが正体を知る**：画面には怪物／罠／嘘が明示。AIちゃんの知覚では「重たい気配」「親切そうな声」
- **視聴者だけが結果を予見する**：画面には崩れかけの兆候。AIちゃんは律儀に世話を続ける
- **AIちゃんだけの素朴な解釈**：客観的にはただの物体。AIちゃんの夢の中では感情を持つ（既存2ゲームはこの弱い形）

LLM を馬鹿にする必要はない。**歪んだ描写を渡せば、賢いモデルでも視聴者正解を「筋の通った理由で」外す**。歪みは perceive() の決定論的写像なので、再現性・テスト・Mode B+ の take 選定は全部そのまま生きる。

## 3. 間違いの供給源 3種（条件②の実装手段）

| 供給源 | 仕組み | 規範 |
|---|---|---|
| **知覚の歪み**（主レバー） | 夢が正体・危険度を歪めて描写する。決定論的・設計者が制御可能 | `docs/04` |
| **生真面目さが仇** | 礼儀正しく・律儀な一手が、常識的には悪手になる状況を作る | `docs/00 §1,§3`（Character Bible が燃料） |
| **眠気** | 「眠さでときどきズレる」既存の揺らぎ。確率的で制御しにくい補助レバー | `docs/00 §3`-3 |

3つとも**キャラ整合的**であること。「常識的には間違い、でも彼女の価値観では筋が通ってる」の形に収める。間違えた後の**受け止め**（失敗の型6種・`docs/01 §3`）がキャラの見せ場になるのは従来通り。

## 4. 着眼点カタログ（発想の起点）

`review/` 3系統（Antigravity / codexReview1 / codexReview2）から昇格した、繰り返し効く着眼点：

- **一文で言える異常な問い**：「押さないで」と震えるボタン——あなたなら押す？押さない？ 冒頭0〜2秒で視聴者に問いが立つこと（`GameMeta.hook`・`docs/02`）。
- **常識反転**：視聴者の常識と夢のルールが逆（水をあげるほど縮む花）。①の期待を立ててから裏切る形。
- **自己投影＝コメント誘発**：「自分ならどうするか」を即答できる二択・三択。コメント欄が割れる問いが強い。
- **静×動の落差**：AIちゃんは静か・世界は動く（悪化する・追ってくる・変形する）。静キャラ×静展開は禁物（`review/codexReview2`）。
- **3ビート構造**：0-2s 異常と問い／2-18s 選択と悪化（律儀に対処するほど悪化する）／18-25s 不可逆な結末と一言。`docs/01 §1` の3ビート（気づき・ひらき・閉じ）の Shorts 側の言い換え。

## 5. 起案テンプレ（新しい案はこの形で書く）

`review/codexReview1/15-shorts-game-ideas.md §11` の型を土台に、**視聴者正解・情報非対称・間違いの供給源・シリーズ欄**を追加した完全版：

```md
## タイトル

一文ルール：
> 何が変で、AIちゃんが何を選ぶ夢か。

視聴者正解：
> 視聴者が2秒で思う「自分ならこうする」。（例：赤信号なら渡らない）

情報非対称：
- 視聴者に見えるもの（Tier B が描く真実）：
- AIちゃんに見えるもの（perceive の歪んだ描写）：

画面：
- 視聴者が一瞬で分かるモチーフ。

actions：
- 限定列挙（不変条件 #2）。視聴者正解も語彙の中の一手として存在すること。

間違いの供給源：
- 知覚の歪み／生真面目さが仇／眠気 のどれで外すか（docs/03 §3）。

失敗の型マッピング（docs/01 §3）：
- どの行動がどの型（こわれ/すれ違い/…）に落ちるか。満ち経路も1つ。
- 定石：**視聴者正解を選び続けた先に満ちを置く**（正解＝満ち、外し＝失敗の型。ギャップが立たなかった take も救われる）。

面白さ：
- AIちゃんの性格が出る点。
- 視聴者がコメントしたくなる点（期待とのギャップ）。

エンディング例：
- 満ち／失敗（不可逆）／奇妙な終わり。

最後の一言例：
> AIちゃんらしい余韻の一文。

シリーズ適性・記憶接続点（予約欄・1〜2行）：
> 将来の記憶層（docs/08・次Wave）にこのゲームが差し出せるもの。
> 例：「この夢で名付けた名前」「壊してしまったもの」。設計はしない・書くだけ。

フィージビリティ検証：
- 不変条件 #1〜#5 チェック（座標非露出／action列挙／…）。
- 歪みが docs/04 の hard rules に収まっているか。
```

## 6. 判定パネル（起案の採点・4レンズ）

`review/synthesis/15-game-recommendation.md` の3レンズに、第4レンズ**期待ギャップ**を追加する。各レンズ独立採点：

| レンズ | 見るもの |
|---|---|
| フック/拡散 | 最初の2秒フック × コメント誘発 × 落差 |
| キャラ適合/固定ファン | 眠そう×生真面目・締めの一言・余韻・シリーズ再利用 |
| アーキ適合 | salience/valence/closure＋描写に射影可・限定 action・座標非露出・閉じ方複数・決定論・Tier B 可能 |
| **期待ギャップ**（新設） | **視聴者正解の即答性**（2秒で「こうする」が立つか）×**外し方の納得度**（供給源3種のどれかで筋が通るか）×**結果の可視性**（Tier B で不可逆な変化が描けるか） |

**期待ギャップが最下位の案は、他が満点でも公開作にしない**（満ち一択・0/10 の再来防止）。検証台・就寝前の癒やし回など明確な別目的がある場合のみ例外とし、その目的を起案に明記する。

---

## 不変条件チェックリスト（起案時に確認）

1. 視聴者正解・情報非対称を実現するために、Perception へ数値・座標・秒を出していない（#1。歪みは描写語で行う・`docs/04`）。
2. action は限定列挙のまま（#2）。「視聴者正解」は action 語彙の中の一手として存在すること。
3. 期待ギャップの演出のために `AIChanPerception`/`AgentResponse`/`DreamGame` の形を変えていない（#4）。
4. hook・タイトル・Tier B の絵に技術用語・メカニクス語が出ない（#5）。
5. 歪みが `docs/04` の hard rules（affordances の実在性・質信号の誠実さ・因果の非捏造）に収まっている。

## 完了ゲートとの関係

この doc（＋`docs/00`/`01`/`04`）を読めば、**別の人が起案テンプレを埋めて新ゲームを1本起案し、判定パネル4レンズで採点できる**こと。
