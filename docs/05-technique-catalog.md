# 05 — Technique Catalog（技法カタログ：目次と共通原理）

このドキュメントは**エンタメ技法の索引**。各技法の詳細は `docs/techniques/*.md`（1技法1ファイル）にある。
`docs/03`（起案の原理：期待ギャップ・ヘイト管理・判定パネル）が「どう考えるか」を定めるのに対し、ここは**「どんな型があるか」の辞典**。起案時は docs/03 → 本 doc → 選んだ技法ファイルの順に読む。

新しい技法を見つけたら（外部ネタ帳を掘るたびに）、`docs/techniques/` にファイルを足して本 doc の表に登録する。**原理（docs/03）は安定、辞典（ここ）は成長**——この変更頻度の分離が本 doc を分けた理由。

---

## 1. 情報関係の5類型（技法の親分類）

面白さを作る情報構造は「視聴者＞AIちゃん（劇的アイロニー）」だけではない。**誰が何を知っているか**の設計が技法の親分類になる：

| 情報関係 | 生まれる面白さ | 代表技法 |
|---|---|---|
| 視聴者 ＞ AIちゃん | 劇的アイロニー。「そこ危ない！」 | 劇的アイロニー |
| 視聴者 ＝ AIちゃん | サスペンス。一緒に考える・間に合うか | サスペンス＋タイムリミット／エスカレーション |
| AIちゃん ＞ 視聴者 | 秘策・天才性・意外な正解。「そういう意味かよ」 | クィブル |
| 双方が不完全 | 謎解き・ルール発見。どちらが先に気づくか | ルール発見／チェーホフの銃 |
| 双方が別の情報を持つ | 解釈対立・議論。コメント欄が割れる | 倫理的ジレンマ／心理戦・ブラフ |

シリーズ全体は、1つの問いではなく**複数の問い**で回す：
**気付けるか／間に合うか／信じてよいか／どちらを選ぶか／ルールを破らず突破できるか／最初の物がどう使われるか／AIちゃんの予想は当たるか**。

## 2. 技法一覧（優先順位つき）

| 順位 | 技法 | ファイル | 一言 | AIちゃんは |
|---:|---|---|---|---|
| 1 | クィブル | `techniques/quibble.md` | ルールの文言を厳密解釈して抜け穴で突破。LLM 固有の言語能力が武器 | **勝つ** |

| 5 | チェーホフの銃 | `techniques/chekhovs-gun.md` | 冒頭の変な物が最後に効く。再視聴を促す | どちらも |

| — | 公平などんでん返し | `techniques/fair-twist.md` | 視聴者もAIちゃんも騙される。**全体の15〜20%の変化球** | 負ける（視聴者も） |

順位は「このプロジェクトとの相性」（LLM の武器を活かせるか・キャラが立つか・25秒で立つか）。**上位＝良い、ではなく配合の目安**：クィブル等の「勝ち回」を混ぜることが、アイロニー偏重（＝AIちゃんの被害者化・`docs/03` ヘイト管理）への構造的な対策になる。

## 3. 1動画1技法

**1本の夢に技法は1つだけ置く。** 「アイロニー＋ジレンマ＋どんでん返し＋伏線回収」と重ねると25秒では説明不足になる。複数試したい組み合わせがあるなら、別々の夢として起案する。

## 4. 25秒5ビート（技法を尺に落とす共通形）

| 秒 | 中身 | docs/01 の3ビート対応 |
|---|---|---|
| 0〜2s | ルール・危険・奇妙な物を一目で提示（hook カード） | 気づき（opening） |
| 3〜7s | AIちゃんが状況を観察 | 気づき（opening） |
| 8〜14s | 判断または推理 | ひらき（unfolding） |
| 15〜20s | 行動と即時の結果 | ひらき（unfolding） |
| 21〜25s | 反転・報酬・失敗・ひとこと | 閉じ（closing・Closing Beat） |

秒数は目安（実際の尺配分は音声長駆動・`docs/11 §5`）。重要なのは**どの技法もこの5拍に落とせるか**を起案時に検査すること。落ちない技法は25秒向きでない（長編向け）。

## 5. 技法→ゲーム変換テンプレ

外部ネタ帳や技法ページを読んだら、ゲームに直行せず**まずこの形に変換**する。埋まらない欄がある技法は採用しない：

```text
技法名：
視聴者が抱く疑問：
AIちゃんが行う選択：
面白さが発生する瞬間：
画面で見える結果：
AIちゃんの性格が原因になる箇所：
25秒で説明可能か：
同じ型を何本作れるか（量産性）：
```

記入例（クィブル）：

```text
技法名：クィブル
視聴者が抱く疑問：この禁止ルールをどう突破する？
AIちゃんが行う選択：ルールを守る／文言の抜け穴を探す
面白さが発生する瞬間：「そういう意味かよ」
画面で見える結果：部屋ごと動く
AIちゃんの性格が原因になる箇所：言葉を額面以上に厳密に読む生真面目さ
25秒で説明可能か：可能
同じ型を何本作れるか：高い（禁止ルールの数だけ作れる）
```

ここを通ったら `docs/03 §6` の起案テンプレ（フル版）に進む。

## 6. 外部ネタ帳の使い方

**シナリオの書き方まとめwiki**（https://w.atwiki.jp/scenario2/ ・約1300ページ）を第一のネタ帳として登録する。

- 位置づけ：**正解集ではなく発想辞典**（アイデア索引として優秀。ただし長編向け技法が多く、Shorts 設計書としてはそのまま使えない）。読んだら§5の変換テンプレへ——**1つの選択と1つの結果に圧縮**できるものだけ採用する。
- 効くページ群：技法系（サスペンス・タイムリミット・クィブル・自己成就型予言・チェーホフの銃・どんでん返し・心理戦・ギャップ）／ヘイト管理（`docs/03` ヘイト管理節の出典）／テーマ・モチーフ系（概念的テーマ・倫理的ジレンマ・思考実験・見るなのタブー・怪異・都市伝説）。
- **神話・民話モチーフを優先**する（見るなのタブー＝鶴の恩返し・イザナミ・ロトの妻の系譜。「押さないでボタン」はこの変奏）。文化的に共有された脚本は**視聴者の期待がプリインストール済み**＝`docs/03 §1` 条件①がタダで手に入り、パブリックドメインで安全。
- **著作権規範：アイデア・モチーフ・構造の利用は可。文章の転写は不可。** 固有作品名・固有キャラは成果物の表側に出さない。
- アクセス注意：wiki 本体は Cloudflare 保護で機械取得不可。エージェントが参照するときは Wayback Machine（`https://web.archive.org/web/*/https://w.atwiki.jp/scenario2/pages/N.html`）か、人間がブラウザで開いて貼り付ける。

## 不変条件チェックリスト（技法をゲームに落とすときに確認）

1. 技法の実現に必要な情報差は、AIちゃん側では**質的描写のみ**で作る（#1・`docs/04`）。視聴者側の情報は Tier B が担う（`docs/11 §2`）。
2. 技法の「仕掛け」（抜け穴・トリガー・伏線）は **action 語彙と RawState の中に実在**させる（#2・決定論）。LLM の speech 内容を apply の入力にしない。
3. 技法のために契約・schema・response の形を変えない（#4）。変えたくなったら doc-first＋人間承認（心理戦の LLM 対話化はこの理由で次 wave 予約）。
4. 技法名（クィブル・アイロニー等）を成果物の表側に出さない（#5）。
5. 1動画1技法（§3）。
