# 能力と課題のギャップ（得意不得意の形が人間と違う）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：普段の印象と意外な能力・弱点の差でキャラを印象付ける。**「AIだから馬鹿」ではなく「得意不得意の形が人間と違う」**。
- **情報関係**：視聴者 ＝ AIちゃん（課題は共有。ギャップは能力側にある）。
- **視聴者の問い**：これ、AIちゃんはどっちだ？（一瞬で解く？　永遠に悩む？）
- **優先度**：順位外（独自追加）。キャラ深掘り回・固定ファン向け。

## 二方向のギャップ

| 方向 | 形 | 例 |
|---|---|---|
| **高度な課題を一瞬で** | 人間が唸る問題を、寝ぼけたまま即答する | 複雑な紋様の規則を一目で読む——のに、ドアの押す/引くを間違え続ける |
| **単純な課題を異常に深く** | 誰でもできる作業で、存在論に潜ってしまう | 「リンゴを箱に入れるだけ」→箱の内と外について考え始める→時間切れ |

## 25秒での形

| 拍 | 例（リンゴを箱に） |
|---|---|
| 0-2s | 机にリンゴひとつ、箱ひとつ。「箱に入れてください」 |
| 3-7s | 「……いれる、って、どこからが『中』なんだろう」 |
| 8-14s | 箱を傾ける・裏返す・中を覗く（真剣） |
| 15-20s | 世界が待ちくたびれて、箱が自分からリンゴを飲み込む |
| 21-25s | 「……はこの方が、はたらきものだった」 |

## 契約との噛み合わせ（要注意）

- ギャップは**プロンプト（キャラを馬鹿に書き換える）でなく課題設計で出す**：`docs/00` の Decision Policy はそのまま。「観察を律儀に言語化する」性質が、単純課題では過剰思考として**自然に**現れる課題を選ぶ。
- 「即answering 側」のギャップは、難しい観察（紋様の規則）を perceive が描写として渡し、LLM が一手で正しく選べる affordance 設計にする——**知性の演出も機構で**。
- 過剰思考ループは `maxTurns`＝醒め（型2）で自然に閉じる（`docs/01`）。**maxTurns 常態化の禁止（docs/09）との緊張に注意**：この型を使うゲームは「時間切れが演出として正しい」例外なので、起案時にその旨を明記して人間承認を取る。
- 世界側の痺れ切らし（箱が自分で飲み込む）は apply の段階進行で（エスカレーションの最小形を借りるが、主役はギャップ）。

## ゲーム例の種

- **リンゴを箱に**（上記）：単純作業×存在論。
- **一目で解ける星図と、開かない引き戸**：同一ゲーム内で両方向のギャップを対にする。
- **完璧な地図を描くが、自分の現在地だけ分からない**。

量産性：**中**。「人間の簡単⇔AIの簡単」の非対称リストが弾倉。擦るとワンパターン化するのでキャラ回として時々。

## ヘイト管理メモ

**勝ちも負けもある回**（というより勝敗の外）。「馬鹿にする」と紙一重の型なので最重要注意：ギャップは**嘲笑ではなく愛でる**トーンで。過剰思考は「真剣さの過剰」として描き、締めの受け止めで本人が一番納得している形に。**「AIのくせに」系の演出・字幕は絶対に作らない**。
