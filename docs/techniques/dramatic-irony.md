# 劇的アイロニー（視聴者は知ってる、本人は知らない）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。
> **この技法の本則は `docs/03 §2`（情報関係）と `docs/04`（知覚の歪みポリシー）**——本ファイルは索引と注意書きのみ。

- **一言**：視聴者には真実（Tier B）が見え、AIちゃんには夢の歪んだ描写しか見えない。「そこ危ない！」の緊張と笑い。
- **情報関係**：視聴者 ＞ AIちゃん。
- **視聴者の問い**：気付けるか？（気付かないまま、どうなってしまうのか）
- **優先度**：**7位**。分かりやすく強力だが、**多用すると AIちゃんが被害者化する**（「毎回騙される子」＝ドッキリがいじめに見える構造・`docs/03` ヘイト管理節）。

## 使いどころ

- 実装の規範は揃っている：歪みの whitelist／hard rules は `docs/04`、定番形3つ（正体を知るのは視聴者だけ／結果を予見するのは視聴者だけ／AIちゃんだけの素朴な解釈）は `docs/03 §2`。
- **配合ルール**：アイロニー回の連投を避け、勝ち回（クィブル）・情報対等回（サスペンス・エスカレーション）と交互に。シリーズ内比率はヘイト管理節（`docs/03`）の頻度管理に従う。
- アイロニーで負けた回の直後は、**同じ夢の要素で「勝つ」続編**（視聴者正解を今度は取る・回収する）を置くと被害者化が中和される——シリーズ適性欄（起案テンプレ）の使いどころ。

## ヘイト管理メモ

**負け回**。この型だけが持つ固有リスク＝**設計者の作為が透ける**（歪みが「罠を仕掛けた」に見える）。歪みは夢の性質として一貫させ（`docs/04` hard rule 4）、失敗の受け止め（`docs/01`）で尊厳を必ず回復する。
