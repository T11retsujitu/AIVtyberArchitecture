# エスカレーション（同じ規則のまま、世界だけ異常になる）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：規則は最初から最後まで同じ。繰り返すたびに**状況だけが段階的に異常化**していく。面白さは「次は何が増えるか」。
- **情報関係**：視聴者 ＝ AIちゃん（情報差なしで成立する。差が要らないのが強み）。
- **視聴者の問い**：次は何が来る？　どこまで行く？
- **優先度**：順位外（独自追加）。情報設計が不要なぶん立ち上げが軽く、**他の技法の骨格にも重ねやすい**（ただし1動画1技法の原則は守る——主役はどちらか1つ）。

## 25秒での形

| 拍 | 例（起こさないで） |
|---|---|
| 0-2s | 「起こさないで」。部屋の奥で何かが眠っている |
| 3-7s | 忍び足。……床が、鳴る |
| 8-14s | 壁が、くしゃみをする。「か、かべ……？」 |
| 15-20s | **AIちゃん自身が眠くなり、寝言を言いはじめる** |
| 21-25s | 何かが、目を覚ます。「……おそろい、だね。おはよう」 |

## 契約との噛み合わせ（要注意）

- 段階は **RawState の進行カウンタ＋apply の純関数**で刻む（乱数なし・決定論）。各段の異常は `scene.elements`（新 ref の追加）と `feedback: strange` で渡す。
- 異常の階段は**単調増加**に固定する（戻ると「ぐるり」型と混線する）。5〜7ターンで頂点＝終端に届く設計（`MAX_TURNS` 内）。
- 最終段に**キャラ自身を巻き込む**と一番おいしい（眠気で寝言＝AIちゃんの体質が最後の一段になる——自己成就型予言との近縁部だが、こちらは避けようがない世界の性質）。
- Tier B は段ごとの異常を**絵の変化**で必ず見せる（画面が動く＝静キャラ×動く世界・`docs/03`）。

## ゲーム例の種

- **起こさないで**（上記）：静けさの規則×騒がしくなる世界。
- **一列に並べたい何か**：整えるたび、並べる対象が増えて大きくなる（最後は部屋自体が列に並びたがる）。
- **相槌を打つ壁**：話しかけるたび相槌が増え、最後は部屋中が会話に参加する。

量産性：**高い**。「規則1つ×異常の階段」の組み合わせ。階段のネタ切れだけが敵。

## ヘイト管理メモ

**勝ちも負けもある回**。世界が理不尽化する分、AIちゃんの落ち度はゼロ——責任配分（世界≫本人）が最初から満たされる安全な型。オチが「彼女のせい」に見えないよう、最終段のトリガーは**体質（眠気）や偶然**にし、判断ミスにしない。
