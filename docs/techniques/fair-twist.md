# 公平などんでん返し（視聴者もAIも騙される）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：視聴者とAIちゃんを**同じ方向へ**誘導し、最後に意味を反転させる。単なる唐突な逆転ではなく、事前の手掛かりと論理的整合性が必須（＝公平）。
- **情報関係**：視聴者 ＝ AIちゃん（両者とも誤導される）。
- **視聴者の問い**：信じてよいか？（見た目通りか？）
- **優先度**：順位外の**変化球**。**全体の15〜20%に限定**——多用すると視聴者が「どうせ見た目と逆なんだろ」と学習し、全ゲームの第一印象が信用されなくなる。

## 25秒での形

| 拍 | 例（怖い怪物とかわいい妖精） |
|---|---|
| 0-2s | 怖い怪物が唸り、かわいい妖精が「こっちへ」と手招き |
| 3-7s | 「……こわい。あっちの子に、ついていこう」 |
| 8-14s | 妖精に従う。怪物はなおも吠える——**出口の方角を指して** |
| 15-20s | 妖精の影が異様に大きい。足元に罠——**妖精が捕食者だった** |
| 21-25s | 「……吠えてたの、とめてくれてたんだ」 |

## 契約との噛み合わせ（要注意）

- **フェアネスの担保は Tier B の責務**：手掛かり（怪物が出口を指す・妖精の影だけ大きい・妖精が罠を避けて歩く）を**絵に必ず仕込む**。見返した視聴者が「言われてみれば」と検証できること。
- `docs/04` hard rule 3（因果の非捏造）との両立：歪めてよいのは**ラベリング**（捕食者→「かわいい妖精」、守護者→「怖い怪物」＝ whitelist の正体の歪み）。**起きる事象は全て真実**（怪物は実際に出口を指している）。事象を捏造した瞬間、どんでん返しは詐欺になる。
- 反転の開示は終端の feedback／Closing Beat（`docs/09`）で行う。`valence` は反転前 neutral/strange → 反転時 bad（または good）と**真実に沿って**動かす。

## ゲーム例の種

- **怖い怪物とかわいい妖精**（上記）：外見と役割の反転。
- **「逃げろ」と言う檻**：檻がAIちゃんを閉じ込めているのではなく、**外の何かから守っていた**。
- **腐った果物と輝く果物**：輝く方が罠。腐った方は「先客が齧って確かめてくれた」印。

量産性：**低〜中**（意図的に絞る）。反転パターンは有限で、擦ると信用を失う。

## ヘイト管理メモ

**AIちゃんも視聴者も負ける回**——「一緒に騙された」ので被害者化しない（視聴者が共犯）。ただし手掛かりが不十分だと「後出し」＝制作側への不信に変わる。**手掛かり3点以上・見返しで全部確認できる**を合格条件に。
