# クィブル（ルールの抜け穴）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：ルールの文言を厳密に解釈し、抜け穴によって意図された制約を「合法的に」回避する。
- **情報関係**：AIちゃん ＞ 視聴者（秘策・意外な正解）。
- **視聴者の問い**：この禁止ルールを、どう突破する？
- **優先度**：**1位**。LLM 固有の言語能力（文言の解釈）で個性を出せる唯一の型。「AIが間違う動画」ではなく**「AIの考え方が人間と違うから面白い動画」**になる——被害者化（`docs/03` ヘイト管理）の根本解。

## 25秒での形

| 拍 | 例（一歩も動くな） |
|---|---|
| 0-2s | 「あなたは一歩も動いてはいけません」。出口は遠くに見える |
| 3-7s | AIちゃんが床を見回す。「動いちゃ、だめなんだ……」 |
| 8-14s | 足元のレバーに気づく。「わたしが動かなければ、いいんだよね」 |
| 15-20s | レバーを引く——**部屋そのものが出口まで動く** |
| 21-25s | 「わたしは、動いてないと思う。……部屋が歩いてるだけ」 |

## 契約との噛み合わせ（要注意）

- **抜け穴は action 語彙内に実在させる**（不変条件 #2）。「創造的な解」を自由文字列で受けることはできない。設計者が抜け穴 action（例：`pull_lever`）を仕込み、affordance の label は**中立に**書く（「床のレバーに触れてみる」——「抜け穴を使う」と書いたら台無し）。
- **発見の知性は LLM の選択＋speech の解釈が担う**。同じ affordances から普通の手（`stay`／`step`）でなく抜け穴を選び、その理由を彼女の言葉で言う——ここがキャラの見せ場。
- 抜け穴に気づかない take も出る。**Mode B+ の選定で「気づいた take」を拾う**（打率レバー：hint の匂わせ・affordance 数を絞る）。
- 抜け穴経由の終端は「満ち」でよい（`docs/01`：珍しい**知恵による満ち**）。

## ゲーム例の種

- **一歩も動くな**（上記）：禁止＋環境操作の抜け穴。
- **喋ってはいけない部屋**：`hum`（鼻歌）や `write`（指で書く）が語彙にある。「しゃべって、ないよ」。
- **触れてはいけない宝物**：`blow`（息を吹きかける）で台座ごと傾ける。

量産性：**高い**。禁止ルールの数だけ作れる（動くな・喋るな・触るな・見るな・眠るな…）。

## ヘイト管理メモ

**AIちゃんが勝つ回**。被害者化リスクなし。むしろ連投すると「毎回ズルで勝つ」に見えるため、素直な回（サスペンス・ルール発見）と混ぜる。抜け穴が屁理屈すぎると生真面目キャラと衝突する——「ルールを破りたい」ではなく**「ルールを正確に守りたい」結果として抜け穴に至る**トーンを守る（律儀さの延長）。
