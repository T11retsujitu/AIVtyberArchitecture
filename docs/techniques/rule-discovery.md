# ルール発見（視聴者とAIが一緒に解く）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：視聴者もAIちゃんも最初は答えを知らない。1手ごとに規則の一部が判明し、どちらが先に気づくかを競う。
- **情報関係**：双方が不完全（謎解き・ルール発見）。
- **視聴者の問い**：規則に気付けるか？　自分はAIより先に分かるか？
- **優先度**：**4位**。視聴者参加とAIちゃんの知性表現を両立できる唯一の型。

## 25秒での形

| 拍 | 例（増えるドア） |
|---|---|
| 0-2s | ドアが3つ。「3回だけ、開けられます」 |
| 3-7s | 「……どれも、同じに見える」 |
| 8-14s | 赤を開ける→**ドアが2つ増える**。「ふえた……？」 |
| 15-20s | 青を開ける→1つ消える。「……そういう、こと？」 |
| 21-25s | 規則を推測して最後の1手。「たぶん、こっち」——当たる／外れる |

## 契約との噛み合わせ（要注意）

- 規則の開示は **feedback の質的描写**で行う（「開けたら、ドアがふえた」＝ valence: strange）。数を数値で言わない（#1。「ふえた／へった」の質で十分伝わる）。
- 規則自体は `apply` の純関数に固定（決定論）。**後出し変更をしない**——視聴者は録画を見返して検証できる（再視聴の動機にもなる）。
- 視聴者を先行させたい場合は **Tier B に規則のヒントを描く**（ドアの色と数の変化が絵で正確に見える）——「双方不完全」を「視聴者やや優位」に傾ける調整。
- AIちゃんの推測は `observation`/`speech` に出る。**正解を強制しない**：外れて醒め（型2）で閉じても「惜しい回」として成立。

## ゲーム例の種

- **増えるドア**（上記）：色ごとに増減規則。
- **返事をする階段**：踏む段の順番に規則がある。
- **懐かない灯り**：近づくと消え、背を向けると点く——規則発見が常識反転を兼ねる。

量産性：**高い**。「1変数の隠れ規則」の数だけ作れる。ただし3手以内に判明する単純さが必須（25秒制約）。

## ヘイト管理メモ

**勝ちも負けもある回**。AIちゃんが先に気づけば「賢い回」、視聴者が先なら「そこ○○だろ！」のコメント誘発——どちらも健全。規則が複雑すぎて誰も気づけないのが唯一の失敗（すれ違い型の空振り）。**1ゲーム1規則・例外は1つまで**。
