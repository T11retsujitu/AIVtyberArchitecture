# 自己成就型予言（避けようとしたから起こる）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：予言や警告を避けようとした行動そのものが、その結果を実現させる。
- **情報関係**：視聴者 ＝ AIちゃん（警告は両者に見える）〜視聴者 ＞ AIちゃん（真のトリガーは視聴者が先に察する）。
- **視聴者の問い**：AIちゃんの予想（対策）は当たるか？
- **優先度**：**2位**。失敗がランダムでも理不尽でもなく、**キャラの長所がそのまま失敗原因になる**——最も上質なキャラクターコメディ。

## 25秒での形

| 拍 | 例（音を立てると来る） |
|---|---|
| 0-2s | 壁の文字：「怪物を恐れた者から食べられる」 |
| 3-7s | AIちゃんが文字を律儀に読む。「こわがったら、だめなんだ」 |
| 8-14s | 自分に言い聞かせる。「だいじょうぶ。わたしは、こわくない……」 |
| 15-20s | 怪物が近づいてくる——**恐怖ではなく、声に反応していた** |
| 21-25s | 「……しー、って、してればよかったのかも」 |

## 契約との噛み合わせ（要注意）

- **トリガーは speech の内容ではなく action に紐付ける**（決定論：`apply(state, action)` は action しか見ない。セリフ本文を状態遷移の入力にすると再現性が壊れる）。「声を出す」を action（`reassure`＝声に出して自分を落ち着かせる／`stay_silent`）として語彙に置き、夢内の「声」はその action が担う。
- Character Bible の Decision Policy「**まず観察を言葉にする**」（`docs/00 §3`-1）を原因化できる稀有な型：観察・発話系の action を彼女は選びがち——それがトリガー。**性格が原因、世界は論理的**。
- 警告文は `scene.elements[].description` で渡す。真のトリガーは描写の外（＝夢の歪み・`docs/04` whitelist「正体・意図のラベリング」）。ただし feedback は誠実に（声を出すたび salience が上がる＝前触れをテレグラフ。`docs/04` hard rule 2）。

## ゲーム例の種

- **音を立てると来る**（上記）：恐怖ではなく声に反応。
- **心配すると枯れる花**：世話（`water`・`check`）するほど弱る。見守りが正解——常識反転との近縁形。
- **落ちると思うと落ちる橋**：確かめる action（`test_plank`）が板を軋ませる。

量産性：**中〜高**。「警告の真の条件」を1つずらすだけで量産できる。

## ヘイト管理メモ

**負け回**——だが被害者化しにくい：原因が彼女の美徳（生真面目・言語化）であり、世界は嘘をついていない（警告は本当だった。読み方がズレただけ）。締めの受け止め（`docs/01` 失敗の型）で「なるほど、そういうことか」の腑落ちを必ず作る。**理不尽な後出しルールにしない**こと——トリガーは初見でも遡って納得できる論理に。
