# サスペンス＋タイムリミット（間に合うのか）

> 共通原理（1動画1技法・5ビート・変換テンプレ）は `docs/05` が正。

- **一言**：危険は分かるが、結果は分からない。「次に何が起こるか／間に合うのか」の感情を伴う疑問で引っぱる。
- **情報関係**：視聴者 ＝ AIちゃん（同じ危険が見えている。一緒にハラハラする）。
- **視聴者の問い**：間に合うか？　どれを選ぶ？
- **優先度**：**3位**。**25秒という尺そのものをゲーム化できる**。AIちゃんを騙す必要がない＝被害者化ゼロで緊張を作れる。

## 25秒での形

| 拍 | 例（眠る怪物） |
|---|---|
| 0-2s | 眠る怪物・扉・鍵が3本。「起きる前に、開けて」 |
| 3-7s | 「……ふぁ。どの鍵だろう」。怪物の寝息が画面で刻まれる |
| 8-14s | 鍵を観察。「この鍵……少しだけ、あったかいかも」 |
| 15-20s | 1本目——外れ。怪物の目がうっすら開く。2本目を差す |
| 21-25s | 開く（or 開かない）。「……せーふ、かな」／「あ。……おはよう」 |

## 契約との噛み合わせ（要注意）

- **タイマー・残り秒は視聴者側（Tier B）にだけ見せる**。AIちゃんには不変条件 #1 の通り数値を渡さず、`closure`（opening→closing）と描写（「寝息が浅くなってきた」＝salience 上昇）に落とす——`docs/07` の `timeLeft: 3 → closure: 'closing'` 例そのまま。
- ターン制（Mode B+）なので**実時間の焦りは存在しない**。緊張は演出側（`docs/11`：寝息のメーター・音・カット割り）が担う。「AIちゃんの思考時間」は編集で緊張に変換する。
- 試行回数の制限（鍵3本・試せるのは2回）は RawState のカウンタで持ち、affordances の減少として渡す（数値は出さない）。
- 失敗＝こわれ（型3）でなく「**間に合わなかった**」型（醒め・手詰まり）にもできる。成功＝満ち。**どちらに転んでも動画になる**のがこの型の強み。

## ゲーム例の種

- **眠る怪物**（上記）：鍵の選択×目覚めの進行。
- **しぼむ足場**：立っている床が少しずつ薄れる。渡る順番の選択。
- **閉まりかけの扉**：夢の出口が閉じていく。持っていく物を1つだけ選ぶ。

量産性：**高い**。「進行する脅威×限られた試行」の組み合わせで量産できる。

## ヘイト管理メモ

**勝ちも負けもある回**。情報差がないので「騙された」感がなく、失敗しても「惜しかった」になる——ヘイト管理上もっとも安全な型。連投しても被害者化しないが、単調化はする（緊張の質が毎回同じ）ため他の型と交互に。
