# AIVtuber向け ゲームアイデア集

『AIVtuber×Shorts×自作ゲーム』というコンセプトと、アーキテクチャ（特に「失敗の型6種」と「25秒構造」）に基づいたゲーム案です。
「勝敗が明確なゲーム」ではなく、「AIちゃんの性格（眠そう×生真面目）と反応のギャップを引き出せるシチュエーション」を作ることを目的としています。

---

## 案1：背後の影（ホラー / だるまさんがころんだ系）

* **コンセプト:** 
  暗い廊下。背後から「何か」が近づいてくる。AIちゃんにはその姿（ピクセルや座標）は見えず、「気配の濃さ（`salience: 'faint' → 'vivid'`）」だけが描写として渡される。
* **行動（action）の語彙:** 
  `立ち止まる` / `少し歩く` / `振り返る` / `目を閉じる`
* **Shortsとしての面白さ:**
  視聴者には恐ろしいバケモノが近づいている映像が見えているのに、AIちゃんは「……なんだか、後ろが重たい気がする」と呑気です。この視覚的緊張感とキャラクターの無反応さのギャップが強烈なフックになります。
* **想定される「失敗の型（閉じ方）」:**
  * **【こわれ】** 振り返ってバケモノと目が合ってしまいゲームオーバー。「あ……見られちゃった。こわい、かも」と静かに終わる。
  * **【見失い】** 目を閉じ続けてやり過ごすとバケモノが消える。「あれ……気配が、なくなった。どこ、いったんだろう」

---

## 案2：こわれやすい光（ASMR / 癒やし系）

* **コンセプト:** 
  目の前に、フワフワと漂う温かい光（蛍や精霊のようなもの）がいる。少しでも乱暴に動くと逃げるか、壊れてしまう。
* **行動（action）の語彙:** 
  `じっと見つめる` / `そっと手を伸ばす` / `追いかける` / `手のひらを開く`
* **Shortsとしての面白さ:**
  AIちゃんの「ふぁ……」という眠そうな声と、美しい光の映像がマッチし、寝る前に見たくなる環境映像（ASMR）として固定ファンがつきます。
* **想定される「失敗の型（閉じ方）」:**
  * **【すれ違い】** 追いかけすぎて逃げられる（`valence: 'strange'`の連続）。「うまく、噛み合わない……わたしのやり方じゃ、ないのかも」
  * **【満ち（成功）】** じっと待ち、手のひらに光が乗る。「……よかった。あったかいまま、閉じるんだ」
  * **【こわれ】** 早く触ろうとしすぎて光が弾ける。「あ……こわしちゃった、かも。そっとしておけば、よかった」

---

## 案3：同じ部屋のループ（シュール / 謎解き系）

* **コンセプト:** 
  左右に扉があるだけの部屋。正しい扉を選ばないと、永遠に同じ部屋に戻ってくる（8番出口のようなループ構造）。
* **行動（action）の語彙:** 
  `右の扉をあける` / `左の扉をあける` / `部屋を見渡す` / `その場に座り込む`
* **Shortsとしての面白さ:**
  Shorts特有の「動画のループ再生」と完璧に相性が良いです。動画が終わって最初に戻ったとき、ゲーム内のループとリンクするような演出が可能です。
* **想定される「失敗の型（閉じ方）」:**
  * **【ぐるり】** 間違った扉を選び続け、進展しないままターン上限（25秒）を迎える。「これ、さっきも見た気がする……ずっと、同じところにいる」
  * **【手詰まり】** どちらの扉も開かなくなる。「……もう、どこにも行けないみたい。ここで、おしまいなのかな」

---

## 設計上のポイント

アーキテクチャ上、AIちゃんには「敵のHP」も「残り秒数」も表示されません。アクションゲームやパズルゲームを作ると、LLMが状況を理解できずにランダム行動になり破綻してしまいます。

しかし、上記3つのような**「気配」「距離感」「不条理」をテーマにしたゲーム**であれば、AIちゃんに渡す情報が「質的な描写（`salience` や `valence`）」だけで十分に成立します。最初の実装（Phase 0）としては、状態遷移がシンプルで映像映えもする**「案2（こわれやすい光）」**か**「案1（背後の影）」**から着手することをおすすめします。
