## 推奨（公開第1作）：**押さないでボタン**

**一文ルール（表側hook）**：「押さないで」と震えるボタンがひとつ。——あなたなら、押す？押さない？

**なぜこれか**：3レンズを最もよく束ねる唯一の候補（フック 9.5＝1位、アーキ適合 9＝1位タイ）。弱点はキャラ適合(6)だが、そのレンズは既存の praise-room(9) がテスト台として担うので分業が成立する。公開第1作は「フック／拡散の顔」を担うべき。さらに最重要オープン問題を正面から潰す：(1)冒頭フック未設計 → 一行で誰もが即答したくなる最強の2秒フック、(2)中盤の反転/落差 → 押す衝動を懇願で裏切る good→strange→bad の落差が設計に内在、(3)自作ゲーム感 → 喋って震える一個のボタンで一目で伝わり Tier-B でも表情化しやすい。docs/01 §4 の「満ち以外の閉じ方を複数」を6種の失敗の型ほぼ全部で満たせる稀有な題材。

**actions（限定列挙）**
- `press` — 「おもいきって、押してみる」：唯一の不可逆トリガ。こわれ(型3)の引き金
- `wait` — 「押さないまま、そばで待つ」：付き添うと震えが静まる＝満ち(受容)への道
- `hover` — 「押さずに、そっと手をかざす」：中途半端でボタンが余計ざわつく＝すれ違い(型4)
- `stepBack` — 「そっと、離れてみる」：離れると気配が追ってくる＝ぐるり(型6)／極まれば手詰まり(型1)
- `lookAway` — 「目を、そらしてみる」：注意をやめると存在感が薄れる＝見失い(型5)

**Perception 写像（生数値ゼロ）**：ボタンの内部スカラーを `salience` 3段＋description に、局面を `closure` に、結果を `feedback.valence`（静まり=good／壊れ=bad／空回り=strange）に射影。距離・秒・カウンタは一切露出しない。

**失敗の型マッピング**
- `press`（震え極まり後） → 型3こわれ（bad / terminal）＝最大の落差ビート。初手 press は怯み警告（strange）で前触れを出し、unfolding を必ず1拍挟む
- 付き添って `wait` → 満ち・受容（good / terminal）＝唯一の非失敗・オチの反転
- `hover` 継続 → 型4すれ違い（strange の連続）
- `lookAway` 継続 → 型5見失い（salience vivid→faint→消滅）
- `stepBack` 継続 → 型6ぐるり（maxTurns）、極まれば型1手詰まり（affordances 空→deadend）
- 尺切れ → 型2醒め（maxTurns フォールバック）

**冒頭フックカード**：「押さないで」と、ボタンがふるえている。
**締めの一言**：満ち＝「……よかった。押さないまま、あなたは眠るんだね」／こわれ＝「あ……押しちゃった。やめてって、言ってたのに」

**フィージビリティ検証（不変条件チェック — 通過）**：座標/秒/スコア露出なし・action は限定列挙・LLM が乱択化する隠れスカラーなし（選択は最適化でなく価値観＝正しさよりらしさ）・seed+action で決定的再生。要注意3点：ⓐ press 即terminalで数秒take → 初手 press は警告にして unfolding を挟む。ⓑ「追ってくる/暴走」を座標で描きたくなる誘惑 → salience:vivid＋描写に固定し `assertNoRawMechanics` でゲート。ⓒ `wait` が満ちに向かうか堂々巡りかを `feedback.valence` で必ずテレグラフ（盲目の50/50にしない）。

## 次点・不採用

**第2作＝こわれやすい光**：ボタンが弱い「癒やし/ASMR/満ち」を補完。満ち経路と Tier-B の環境映像を検証。就寝前固定ファン向けの余韻。
**検証用後続＝背後の影**：「視聴者は怪物／AIちゃんは呑気」の静×動フォーマットの横展開検証（ホラーは癒やしと逆・キャラ適合最下位なので第1作にはしない）。

**不採用（第1作に向かない）**
- **コメントで育つ夢の庭**：状態が外部コメント依存で `init(seed)` 決定論を満たせず、単語→生き物の生成が action 列挙不能。アーキ非適合。
- **出口が引き止めてくる迷路**：本質的に空間ナビ／パスファインディングで座標必須。座標非露出契約下で乱択破綻の典型。
- **忘れた猫の倉庫 / 捨てるものを選ぶ部屋**：キャラ適合は高いが、名前候補・抽象名詞が内容依存で語彙が肥大（実質自由文字列化リスク）、抽象名詞の Tier-B 可視化が難しく、冒頭フックも弱め。retention 中盤回として後続。
- **片付けられたい机 / 世話をすると嫌われる生き物**：複数オブジェクトの個別選択や成長スカラーが salience 3段を超えて多軸化、action 語彙も未確定で乱択化リスク。
