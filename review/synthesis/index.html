推奨（公開第1作）：押さないでボタン 実装済

「押さないで」と震えるボタンがひとつ。——あなたなら、押す？押さない？

なぜ：フック9.5（1位）＋アーキ適合9（1位タイ）。弱点のキャラ適合(6)は praise-room(9) が検証台として分業。P0②フックを直撃し、押す衝動を懇願で裏切る good→strange→bad の落差が内在。docs/01 の失敗の型6種ほぼ全部を出せる稀有な題材。

action意味閉じ方
pressおもいきって押す型3こわれ（初手は怯み警告→2手目で破壊）
waitそばで待つ満ち（受容・唯一の非失敗）
hover手をかざす型4すれ違い
stepBack離れる型6ぐるり→型1手詰まり(deadend)
lookAway目をそらす型5見失い（salience vivid→faint）

締めの一言：満ち＝「……よかった。押さないまま、あなたは眠るんだね」／こわれ＝「あ……押しちゃった。やめてって、言ってたのに」
不変条件：座標/秒/スコア露出なし・action 限定列挙・seed+action で決定的再生 ✓

次点・不採用（クリックで展開）

第2作＝こわれやすい光：癒やし/ASMR/満ちを補完。検証用＝背後の影：静×動の横展開（ホラーは癒やしと逆で第1作にはしない）。

不採用：コメントで育つ夢の庭（決定性×）／迷路（座標必須）／忘れた猫・捨てる部屋（語彙肥大・抽象名詞の可視化難）／片付け机・世話生き物（多軸スカラーで乱択リスク）。

打率改善プラン docs/14（検証済）

Mode B+ が見つけた 打率0% の根本原因を潰す3本立て（design＋反対弁護レビュー済）。不変条件（数値非露出・限定 action・構造化出力・決定論）は崩さない。詳細は docs/14。

A. ゲーム側で落差
緊張を作る

周囲が「押して」と囁く第2の気配＋見守り継続で震えが増幅。wait を満ちに直結させず press/stepBack の決断を誘発。

B. prompt で反復・能動性
単調を崩す

直近発話を渡し状態語の再利用禁止・「ふぁ」制限・語尾分散・能動 action を最低1回。契約は additive。

C. 日本語ゲート＋再測
不良を弾く

生成後に非文/誤変換を軽量検出し採点前に除外。A/B前後で同条件バッチ→打率を再測。

実装順序：C（計器）→ A → 再測で press 発火を確認 → score-batch＋合否 → B（doc-first 承認後）。効果側を確認してから計器を厚くする。

合格基準（再測で同時に満たす）：直接投稿可 ≥ 3/10／press 終端 ≥ 1/10（stepBack と分離した独立指標）／満ち占有 ≤ 6/10 かつ 満ち率 > 0（退避一択への付け替え防止）／endReason 種別 ≥ 3／antiRepeat 平均 ≥ 3.0（B の実効性ゲート・未達なら B 撤回）／suspect フラグ率 ≤ 1/10。

※ 判定AIが「closingターンにactionがある＝矛盾」と誤認した指摘があったが、実際の締めビート trace.closing は action なしで正しいため、その改修案は不採用。
